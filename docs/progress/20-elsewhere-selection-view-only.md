# 20 — elsewhere-selection-view-only

사용자 보고: Slack류 메신저에서 인풋이 아닌 채팅 텍스트 선택 시(인풋 미포커스로
보임) 결과창이 사라지고 결과가 인풋에 들어가버림.

## 진단 — 라이브 Slack 백그라운드 프로브 (PID 직접 질의, 포커스 무접촉)

- Slack은 채팅 텍스트를 선택해도 **컴포저(입력칸)가 AX 포커스를 유지**한다.
  실측: 포커스 요소 = AXTextArea, settable(value/selectedText) 둘 다 true,
  선택 범위 (0,0), value `"\n"` (len 1).
- 기존 캡처: 컴포저 selectedText 빔 + 범위 0 + **value가 "\n"이라 "full 비었으면
  ⌘C" 조건도 불통** → 채팅 선택이 통째로 누락 → hasSelection=false → 편집 가능 +
  선택 없음 + immediate = 삽입 모드 → 패널 숨고 응답이 컴포저에 타이핑.
  증상("결과창 사라짐 + 인풋에 결과")과 정확히 일치.
- 해결 열쇠: **조상 AXWebArea가 문서 전체 선택을 노출**한다 (Chrome 실측:
  본문 선택을 selectedText로 반환. Slack 웹 영역도 속성 지원 확인 — 선택 없을 땐
  -25212 NoValue, 미지원 -25205가 아님). 컴포저 → 웹 영역 거리는 15홉.

## 변경 (captureContext)

포커스 요소에 선택이 없을 때(범위 0) 분기 재구성:
1. 범위 > 0 → ⌘C 폴백 (기존: 요소 안 선택인데 텍스트만 안 주는 웹뷰).
2. **조상 웹 영역(≤25홉)의 selectedText가 비어있지 않으면 → "포커스 밖 선택"**:
   보기 전용 컨텍스트 반환 (isEditable=false, selectedText=문서 선택,
   axElement=웹 영역). 선택이 포커스 요소 밖이므로 삽입 대상이 아니다.
   axElement를 웹 영역으로 주면 패널 앵커도 선택 위치(텍스트마커 bounds,
   progress/19)에 붙는다.
3. full == nil(AX 침묵) 또는 빈 요소 → ⌘C 폴백 (기존 유지. full이 차 있으면
   폴백 안 함 — 줄 복사 에디터 오탐 방지도 유지).

한계(의도): 웹 영역이 selectedText를 안 주는 앱에선 포커스 밖 선택이 누락돼
삽입 모드로 감 — 줄 복사 오탐 방지와의 트레이드오프, 사례 나오면 재검토.

## 검증

- 라이브 Slack/Chrome 실측: 위 진단 항목 전부. 컴포저→웹영역 15홉, 25홉 상한.
- `swift build` 경고 0, `make` 성공. LLMCore 무변경.
- GUI 실측 필요:
  1. Slack 채팅 텍스트 선택(컴포저 비움) → 핫키 → 선택 미리보기 + 결과가 패널에
     남고, 컴포저에 아무것도 안 들어가는지. 패널 위치가 선택 근처인지.
  2. Slack 컴포저에 직접 타이핑 삽입/컴포저 내 선택 교체 — 기존대로.
  3. 회귀: Chrome 본문 선택 보기 전용, 네이티브 앱 교체/삽입.
